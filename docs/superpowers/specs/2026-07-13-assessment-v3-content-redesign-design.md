# AI能力与风格测评 V3 内容重构设计

## 目标

将现有按五类岗位、每类22道题的 `assessment-v2` 升级为更适合新学员入学摸底的 `assessment-v3`。新版重点识别学员是否只停留在大模型对话、是否会使用 Agent 与工作流、是否完成过真实 AI 项目，以及其稳定的 AI 协作风格。

新版仍是课程起点画像，不是标准化心理测验，不设置个人排名。

## 岗位

岗位调整为四类，并在选择框中直接说明职责：

1. 顾问：负责销售、招生、咨询。
2. 教练：负责学生服务、跟盯、学习督查、作业跟进、学习管理等。
3. 教师：负责授课、教学、教研、课程设计。
4. 通用测评：适用于其他岗位及日常办公场景。

内部岗位代码使用 `consultant`、`coach`、`teacher`、`general`。历史 `assessment-v2` 数据不改写，新答卷保存 `assessment-v3`。

## 测评结构

每位学员完成18道选择题和1道提示词开放题，预计8至10分钟。

### 第一部分：AI基础与项目经历

共5题，了解：

- AI使用频率与持续时间。
- 常用大模型，包括 DeepSeek、豆包、Kimi、通义千问、腾讯元宝、ChatGPT、Claude、Gemini 等。
- Agent、工作流与自动化工具使用情况，包括扣子、Dify、n8n、飞书智能伙伴、Cursor/Codex/Claude Code 等。
- 实际使用AI完成过的任务类型。
- 真实项目及完成程度，包括个人网站、知识库、自动化工作流、小程序等。

工具题允许多选，但不按勾选数量直接累加。计分采用“最高使用深度＋有限广度”，避免全部勾选刷分。

### 第二部分：AI应用方式

共4题，评估学员如何：

- 定义任务与交付目标。
- 准备背景资料和上下文。
- 与AI分轮协作和修正。
- 验证事实、结果和交付质量。

### 第三部分：AI使用风格

共6题，两题判断一条轴，不计能力分：

- 探索 E / 执行 D。
- 委派 A / 共创 C。
- 敏捷 F / 审慎 V。

每条轴继续使用2:1题目权重，避免平局，并输出倾向强度。

### 第四部分：岗位业务场景

共3题。四个岗位分别使用自己的真实场景：

- 顾问：家长咨询准备、异议处理、咨询跟进与信息核验。
- 教练：学情跟盯、作业反馈、学生异常识别与家校沟通。
- 教师：备课授课、作业评价、教研分析与课程设计。
- 通用测评：汇报材料、资料整理、跨部门协作与日常办公交付。

业务题数量保持较少，只用于验证学员能否把通用AI能力迁移到真实工作。

### 第五部分：提示词开放题

共1题。根据岗位给出不同真实任务，要求学员写出会直接交给AI的完整提示词。DeepSeek继续按对象、目的、输入资料、执行方式、输出格式、限制边界、验收标准七项分析。

## 计分与等级

12道基础、应用和业务选择题占能力分60%，开放题占40%；6道风格题不计能力分。

选择题内部按三类归一化计算：

- AI基础与项目经历：30%。
- AI应用方式：45%。
- 岗位业务场景：25%。

等级阈值保持：

- 0–39：AI体验者。
- 40–59：任务表达者。
- 60–79：协同交付者。
- 80–100：Agent推动者。

### 工具深度

只使用对话式大模型属于基础层。能实际使用 Agent、工作流、自动化工具完成任务时获得更高分。仅听说、注册或体验工具不等于掌握，题目通过使用频率、实际任务和项目完成程度交叉判断。

### 项目加分

真实完成个人网站、知识库、自动化工作流等项目时，在固定公式基础上给予项目加分；总分最高100。加分依据完成程度而非项目名称：体验、协助、完成核心功能、运行或上线分别对应不同强度。

### 小程序等级提升

只有同时满足“本人实际参与核心功能开发”和“项目成功运行或上线”时，最终成长等级提升一级，最高提升至L4。该规则在常规总分和等级计算完成后执行，报告中明确标注“真实项目加速项”，不伪造原始分数。

## 选项防暗示规则

新版所有单选行为题必须遵守：

- 同一题选项长度尽量接近，避免最佳选项明显更长。
- 四个选项均为现实中可能出现的合理行为，不使用明显错误、敷衍或带贬义的表述。
- 高分行为只体现方法成熟度，不堆叠过多专业术语。
- 最佳选项位置在题库中均匀分布，前端继续随机打乱展示顺序，服务端按选项ID计分。
- 选项之间一次只区分一个主要变量，例如工具深度、任务定义、资料处理、迭代方式或验证强度。
- 多选工具题不能通过“全选”获得最高分，存在使用深度和项目证据的上限约束。

## AI使用风格图鉴

新增完整图鉴，展示八种风格：

- EAF 灵感探路者。
- EAV 洞察研究者。
- ECF 灵感共创者。
- ECV 深度共研者。
- DAF 敏捷执行者。
- DAV 标准交付者。
- DCF 协同推进者。
- DCV 质量守门人。

每张风格卡包含：典型AI使用方式、核心优势、常见盲区、擅长任务、最佳协作方式和一条升级建议。

图鉴入口放在测评首页和个人报告。教师看板的8型分布提供图鉴入口，但公开投屏仍不显示个人身份。

## 个人报告中的风格分析

个人报告展示：

- 风格代码、名称与一句话总结。
- 三条风格轴的倾向和强度。
- 启动任务的典型方式。
- 与AI分工的典型方式。
- 处理速度与质量的典型方式。
- 三项优势和两项风险。
- 推荐AI工作流程。
- 下一步适合尝试的 Agent 或真实项目类型。

风格文案以工作行为为基础，不使用人格诊断或心理疾病相关表述。

## 数据与兼容

- 新版本标识为 `assessment-v3`。
- 新答卷保存岗位代码、工具多选、项目完成程度、小程序升级标记、原始分数、最终等级与风格轴结果。
- 历史V2答卷继续使用旧规则解释，不批量重算。
- DeepSeek请求仍只发送开放题文本，不发送姓名、岗位、班级、工具选择或项目经历。
- Supabase后端密钥、CORS限制和失败回退策略保持不变。

## 验证

自动测试至少覆盖：

- 四种岗位均生成18道选择题，结构为5＋4＋6＋3。
- 工具多选的深度和广度上限，不能靠全选刷高分。
- Agent实际使用得分高于只使用大模型。
- 网站、知识库、自动化项目按完成程度正确加分。
- 小程序只有满足两项条件才提升一级，L4不越级。
- 选项高分位置分布不固定，展示乱序不影响计分。
- 八种风格与三轴强度全部可生成。
- 个人报告包含完整风格分析，图鉴包含八张风格卡。
- 旧版浏览器密钥配置不会重新出现，前端包不包含供应商密钥。
- 本地预览和GitHub Pages子路径下的场次、报告及图鉴链接均有效。
